Stock Alert System

Dec 11, 2020
Photo by Anna Nekrashevich on Pexels

Overview

Each team (4 students) was tasked with an open-ended prompt to create a back-end REST API and front-end site which saves and retrieves data with Amazon DynamoDB. Pre-made back-end and front-end architecture and automated testing workflows were provided. Teams were given four weeks with weekly sprints.

My Role

In addition contributing to front-end and back-end development, I was the SCRUM master and lead designer.

Problem Statement

With stock prices fluctuating constantly, it is difficult for amateur investors to know when a stock falls below a specific price. Project Jupiter is a stock alert system that will provide price tracking and notification functionality to meet an amateur investor’s needs. Customers to the site will be able to 1. Check a stock’s historical price history and 2. Receive notifications on stocks of interest.

User Stories

As a Jupiter customer…

  • I want to view the price history of a stock on the site, so that I can make a more informed purchasing decision.
  • I want to add to an interest list, so that I can easily access the price history for stocks I am interested in.
  • I want to remove a stock from my interest list, so that I do not have to see stocks I am no longer interested in.
  • I want to view all of the stocks I am tracking on one page, so that I can easily access the price history for the stocks I am interested in.
  • I want to see what are the most tracked stocks on the site, so that I may also track them, if I am interested.
  • I want to receive an alert when a stock of interest drops below my alert price, so that I know immediately that I can purchase the stock below my ask price.

As a Jupiter developer…

  • I want to update an item’s price, so that I can build the price history for an item.

Out of Scope

Key features out of scope include:

  • alerts outside of site (ex. text messages, iPhone notifications, e-mails)
  • price granularity beyond daily closing price

Architecture

Built with Amazon API Gateway and AWS Lambda, the back-end REST API consisted of 7 endpoints coded in Java:

  • GetPriceHistory
  • GetTopStocks
  • GetPriceAlerts
  • UpdateStockPrice
  • UpdateUserAlert
  • DeleteUserAlert
  • GetAlertListByUser

The front end used Google’s Material Design Lite framework.

Table Design

Our service used 3 DynamoDB tables.

Stocks

  • symbol // partition key, string
  • company // string
  • description // string

PriceHistory

  • symbol // partition key, string
  • timestamp // sort key, string
  • close // number

UserAlerts

  • userId // partition key, string
  • symbol // sort key, string
  • alertPrice // number
